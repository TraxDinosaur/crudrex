# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in CRUDREX, please report it responsibly.

### Reporting

**Do not** open a public issue for security vulnerabilities.

Instead, please send an email to:

- **traxdinosaur@users.noreply.github.com** (via GitHub)

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if known)

### Response Time

We will endeavor to:

- Acknowledge your report within 48 hours
- Provide a detailed response within 7 days
- Fix the issue and release a patch as soon as possible

## Supported Versions

Security updates are provided for the latest version of both Python and JavaScript implementations.

## Scope

This policy covers:

- The CRUDREX web application
- API endpoints
- Data storage and access
- Any official distribution channels

## Best Practices for Users

To use CRUDREX securely:

1. **Do not expose to the public internet** - CRUDREX is designed for development/testing, not production use
2. **Use in local environments only** - Keep it on localhost or private networks
3. **Don't store sensitive data** - Avoid storing passwords, API keys, or personal information
4. **Keep updated** - Use the latest version to receive security patches
5. **Review JSON data files** - Regularly review what data is being stored

## Disclosure

We follow responsible disclosure practices:

- Security issues are fixed before being disclosed publicly
- Credit is given to reporters (with permission)
- Security advisories will be published with each fix
